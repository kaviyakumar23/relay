# Relay — Video Script & Submission Playbook

> **Track:** Slack Agent for Good · **Runtime target:** 2:50 (hard ceiling 3:00) · **Tone:** documentary, not a sales reel — the judge should finish *moved*, not just impressed.
> **The one rule that governs everything below:** for a *for-Good* track, **visible integrity is production value.** No real disaster footage, no stock images of suffering (copyright risk *and* exploitative — judges are primed to notice). Restraint reads as respect; respect reads as credibility.

---

## 0. The 30-second promise (why the hook is front-loaded)

Judges mostly watch the first 30–60 seconds. So the film is built so that **if a judge stops at 0:60, they have already seen the whole thesis**: the hook, the Tamil–English message no other team can demo, and one raw plea becoming a structured need that *admits what it doesn't know*. Everything after 1:00 is the emotional payoff for judges who stay.

**If you record nothing else, record 0:00–1:05 perfectly.**

## 2. The two "breathing" beats are the whole movie

Everything else is scaffolding around these two frames. Protect them in the edit:

- **Tension (shot 8):** the drift alert. Hold the frame, kill the VO for a beat, one caption: *"A promise is about to break. A family is waiting."*
- **Release (shot 11):** the recipient's ✅. Warm, brief silence, let it land.

The contrast between them *is* the emotional architecture. A judge feels the difference before they can articulate it.

---

## 3. Honesty overlays (put these on screen wherever they apply)

- `🧪 Relay Simulator · all data fictional` — any time simulated messages post.
- `SLA timers compressed for demo` — any time a clock/drift fires faster than real life.
- `Simulated — both runs measured` — **required** if you show the counterfactual stat card.
- Relay assists volunteer coordinators; **it is not an emergency service.** (Say it once, or caption it once.)

For this track, these overlays don't weaken the film — they *are* the production value. They signal you can be trusted with something that matters.

---

## 4. The numbers — use only what your own runs print

Never speak a number the repo can't reproduce. Before recording, run these and read the actual output:

- `npm run eval` → extraction accuracy, critical-severity recall, contact/locality accuracy.
- `npm run demo` → needs created, deduped, the one evidence-verified close, sitrep recount.
- `npm run counterfactual` → the simulated group-chat-vs-Relay delta (label it **simulated** on screen).

**Honest hero framing** (what the shipped demo actually proves): every intake message becomes a tracked need; duplicates are caught; **one** promise is driven all the way to an evidence-gated, human-signed close; the sitrep's headline figures equal an independent recount; the donor report contains zero seeded phone numbers (grep-verified). Don't claim "N verified deliveries" if your run verified one — "watch *one* promise go all the way" is more honest *and* more cinematic than a big round number.

**Strongest single frame available to you:** the counterfactual delta — a naive group chat leaves needs unclaimed and double-served with nothing proven, while Relay loses none and proves delivery. It's a *measured* number from actually running both simulators. Show it with `Simulated` on screen and it's unbeatable; show it without the label and you've undercut the film's entire thesis.

## 5. How to record each shot (practical)

1. **Drive the story from the judge runner, not by hand-typing.** In `#judges-start-here` press **▶ Run flood demo** — it plays intake → triage → assign → drift → reassign → deliver → sign-off → close as the 🧪 Simulator, so your capture is clean and repeatable. Press **↺ Reset** and re-run until the take is perfect.
2. Screen-record at **1080p minimum**, hide personal DMs/notifications, use a fresh Slack theme (light or dark — pick one and stay).
3. Record **silent**; write the VO to the picture afterward. Muted-first design: the film must read with the sound off (captions carry it), and *land* with the sound on.
4. VO: one human voice, calm, slightly slower than feels natural. Record 3 takes of each line; pick the one that *witnesses* rather than *sells*.
5. Music: sparse, low, and out entirely under the two silence beats (shots 8 and 11). If in doubt, use less.
6. **Legal:** no copyrighted music, no third-party trademarks/logos beyond the tech you actually use, no real footage of real events. Keep an attribution note for any asset.

## 6. If you get the real coordinator testimonial (optional, outranks almost everything)

A genuine Chennai volunteer coordinator, 10–15 seconds, plainly stating the problem in the first person, slots in at **~2:20** (between shots 12 and 13) and becomes the most persuasive frame in the film. Lived experience stated plainly outclasses any statistic. No melodrama — one true sentence.

## 7. Gallery card (thumbnail + tagline)

- **Thumbnail = the transformation, not a logo.** Split frame. Left: a raw panicked message (*"Velachery la 3 families terrace mela irukanga… food venum urgent"*). Right: the structured card — **🔴 CRITICAL · 3 families · Velachery**. It must be readable at postage-stamp size and pose the question *"how?"* — which a click answers. Run the **phone-screen test**: if the CRITICAL chip and the Tamil subtitle are legible on a 6-inch display, ship it.
- **Tagline:** *"In a disaster, the deadliest thing is a lost message."* (face-first, zero feature words). Optional second line: *"Every plea heard. Every promise proven."*
- **First sentence of the description = the same hook.** Never "Relay is a Slack agent that…".

## 8. Writeup craft (the story page)

Most of this already lives in `docs/SUBMISSION.md`; these are the additions that convert a skimming judge:

- **"Try it yourself in 90 seconds" — at the very top:** join the workspace → open `#judges-start-here` → **▶ Run flood demo** → watch 14 messages become a coordinated operation → then the kicker: *"Now type your own plea for help — any language — into `#relay-intake` and watch it become a need."* An interactive judge is a convinced judge.
- **Screenshots as a captioned comic strip, in story order:** *A plea arrives → Relay structures it and admits what it doesn't know → a volunteer promises → the promise drifts → Relay reroutes → delivery proven → the donor report shows receipts.*
- **"By the numbers" block:** 538 hermetic + 17 integration tests · extraction accuracy & critical-severity recall from your eval set · needs verified vs. lost · PII leaks in generated reports: **zero (grep-verified)**.
- **Mini-FAQ, pre-empting the three objections:**
  - *"Is this real disaster data?"* → Fictional, labeled, modeled on floods we lived through.
  - *"What if the AI is wrong — or down?"* → Per-field provenance, human gates on every consequential transition, deterministic fallbacks with no API key. The system degrades *honestly*.
  - *"Doesn't crisis software already exist?"* → Ushahidi/Sahana map crises from *outside* the conversation; nothing tracks whether the promised help actually arrived. Relay closes on **proof, not vibes.**
- **Badge the three qualifying technologies in color on the architecture diagram** so an AI-assisted judging pass cannot miss Slack AI, RTS, and MCP. *(The diagram at `docs/architecture.png` / `relay-crisis-site.vercel.app/architecture.png` already shows all three as labeled rows with logos.)*
- **The Chennai grounding belongs in *Inspiration*** — first person, three sentences, no melodrama.

## 9. The meta-move (closing line for the writeup + video description)

Relay's thesis is *no message gets lost* — so dare the judge to try to lose one:

> *"In a disaster, the deadliest thing is a lost message. Open the sandbox and send one — any language, any panic, any format. Relay will not lose it."*

A judge who types a plea in their own language and watches it become a tracked, verified promise doesn't score your project in the deliberation room — they **defend** it.
